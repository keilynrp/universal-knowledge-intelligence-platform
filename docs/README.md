# Documentation Map

Mapa oficial de documentacion de UKIP.

## Capas documentales

La documentacion del proyecto se divide en dos capas:

### 1. Capa operativa

Es la fuente de verdad para planificar, ejecutar y cerrar trabajo.

Documentos principales:

- `docs/DELIVERY_OPERATING_SYSTEM.md`
- `docs/DOCUMENTATION_GOVERNANCE.md`
- `docs/product/README.md`
- `docs/product/PROGRAM_BACKLOG.md`
- `docs/product/TRACEABILITY_MATRIX.md`
- `docs/product/templates/*`
- `docs/TECHNICAL_ONBOARDING.md`

Enterprise readiness:

- `docs/product/ENTERPRISE_CONTROL_REGISTER.md`
- `docs/product/ENTERPRISE_READINESS_PROGRAM.md`
- `docs/product/epics/EPIC-018-enterprise-assurance-and-operational-readiness.md`
- `docs/product/TRACEABILITY_MATRIX.md`

### 2. Capa historica y de referencia

Preserva memoria institucional, decisiones base, evaluaciones y material de contexto.

Documentos principales:

- `docs/reference/README.md`
- `docs/reference/HISTORICAL_REFERENCE_INDEX.md`
- `docs/EVOLUTION_STRATEGY.md`
- `docs/ARCHITECTURE.md`
- `docs/UKIP_ENTERPRISE_ROADMAP.md`
- `docs/SCIENTOMETRICS.md`
- `docs/ukip_system_evaluation.md`
- otros documentos especificos de propuesta o analisis

## Regla de prioridad

Si existe conflicto entre documentos:

1. prevalece la capa operativa
2. la capa historica se usa como contexto
3. la discrepancia debe resolverse en la documentacion viva
